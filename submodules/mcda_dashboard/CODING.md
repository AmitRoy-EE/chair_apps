# CODING.md 
Some notes on how to style and structure your python code for this project. A unified coding style throughout the project enhances the readability and simplifies code reviews and future development by others.

# code style
There is an official Python coding style guideline by the Python foundation which can be found here:
[PEP 8 -- Style Guide for Python Code | Python.org](https://www.python.org/dev/peps/pep-0008/)
A lot of topics are not clearly defined in this guide or are simply unnecessary. The most important sections of the guide are the ones concerning [comments](https://www.python.org/dev/peps/pep-0008/#comments) and [naming conventions](https://www.python.org/dev/peps/pep-0008/#naming-conventions). Please read these sections carefully before writing code, as they contain many useful facts about python which can be especially useful for more inexperienced python developers.
In this project the `lower_case_underscore` naming style is used wherever applicable according to the guideline.

# Don't try to reinvent the wheel
A general rule of thumb should be to google for at least 5 minutes, before starting to implement. Most of the time, there are good libraries out there, that do precisely what we want regarding the smaller steps towards reaching our goal. Hence, the real challenge for us lies more in finding ready-to-use software, and implement it in a way that serves our project.
